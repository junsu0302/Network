# 네트워크의 기초 용어와 기능

## 1. 네트워크 관련 기초 용어

### 1.1. 네트워크 기초 용어

- **네트워크**
```markdown
전송 매체를 매개로 연결되어 데이터를 교환하는 시스템
```

- **인터페이스**
```markdown
시스템과 전송 매체의 경결 지점에 대한 규격
```

- **프로토콜**
```markdown
시스템이 데이터를 교환할 때 따르는 통신 규칙
```

- **표준화**
```markdown
서로 다른 시스테의 연동 형식 통일
```

#### 1.1.1. 네트워크 구셩 요소

- **시스템**
```markdown
내부 규칙에 따라 능동적으로 동작하는 대상
```

- **인터페이스**
```markdown
1. 시스템과 시스템을 연결하기 위한 표준화된 접근 방식
2. 시스템 간의 연동을 위한 논리적인 규격
3. 두 시스템을 연동하기 위한 접속 지점을 의미하고도 함
4. 시스템 간 동등 or 상하 관계 사이 규칙
```

- **전송매체**
```markdown
1. 시스템끼리 정해진 인터페이스를 연동해 데이터를 전송하기 위한 물리적 전송 수단
2. 인터페이스의 구현 및 동작
```

- **프로토콜**
```markdown
1. 상호 연동되는 시스템이 전송 매체를 통해 데이터를 교환할 때 따르는 규칙
2. 동등한 위치의 시스템 사이 규칙
3. 시스템 간 주고받는 정보의 형식과 절차 순서
```

- **네트워크**
```markdown
1. 통신 매체를 공유하는 여러 시스템이 프로토콜을 사용하여 데이터를 주고받는 단위
2. 네트워크는 라우터(중개 장비)를 사용하여 연결
```

- **인터넷**
```markdown
1. 전 세계 모든 네트워크가 유기적으로 연결되어 동작하는 통합 네트워크
2. 데이터 전달 기능에 한해서 공통으로 IP 프로토콜 사용
```

- **표준화**
```markdown
서로 다른 시스템의 상호 연동을 위한 연동 방식 통일
``` 

### 1.2. 시스템 기초 용어

#### 1.2.1. 시스템의 구분

- **노드**
```markdown
데이터를 주고받을 수 있는 모든 시스템
```

- **호스트**
```markdown
1. 컴퓨팅 기능이 잇는 시스템
2. 사용자가 네트워크에 접속하는 창구
3. 호스트는 클라이언트와 서버로 구분
```

- **클라이언트**
```markdown
서비스를 이용하는 시스템
```

- **서버**
```markdown
1. 서비스를 제공하는 시스템
2. 클라이언트보다 먼저 실행되어 요청 대기
```

## 2. 네트워크의 기능

컴퓨터 네트워크는 물리적으로 **호스트 시스템**과 **전송 매체**로 구분한다. 뿐만 아니라 내부 기능을 기준으로 **계층 모델**로 구분한다.

### 2.1. 계층 모델
ISO(국제 표준화 단체)에서 여러 호스트를 연결하기 위해 연결 방식을 표준화하여 **OSI 7계층 모델**제안 

#### 2.1.1. OSI 7계층 모델

| **응용 계층** | **7 Application Layer** |
| :-: | :- |
| **표현 계층** | **6 Presentation Layer** |
| **세션 계층** | **5 Session Layer** |
| **전송 계층** | **4 transport Layer** |
| **네트워크 계층** | **3 Network Layer** |
| **데이터 링크 계층** | **2 Data Link Layer** |
| **물리 계층** | **1 Physical Layer** |

- **데이터 송수신**
```markdown
1. 일반 사용자가 응용 계층을 통해 데이터 송수신 요청
2. 물리 계층을 통해 상대 호스트에게 전송
3. 수신 호스트가 데이터 수신 및 처리
```

#### 2.1.2. OSI 7계층 모델의 계층별 기능

- **물리 계층**
```markdown
호스트를 전송 매체와 연결하기 위한 인터페이스 규칙과 전송 매체의 톡성을 다룸
```
- **데이터 링크 계층**
```markdown
1. 물리적 전송 오루 감지를 통한 오류 인지
2. 오류 제어
```

- **네트워크 계층**
```markdown
1. 데이터 송수신 과정에서의 올바른 경로 선택 지원
2. 네트워크 부하 증가 시 특정 라우터에서 혼잡 발생
3. 혼잡제어
```

- **전송 계층**
```markdown
1. 송수신 프로세스 간의 연결 기능 제공
2. 프로세스 간의 안전한 데이터 전송 지원
3. 데이터가 전송되는 양 긑단 사이의 연결이 완성되는 곳
```

- **세션 계층**
```markdown
1. 응용 환경에서 사용자 간의 대화 개념의 연결
2. 전송 계층에서 제공하는 연결의 개념과 유사한 세션 연결 지원
```

- **표현 계층**
```markdown
1. 정보를 교환하는 시스템이 표준화된 방법으로 데이터를 인식
2. 압축을 통해 전송 데이터 양 감소
3. 암호화를 통해 데이터 보호
```

- **응용 계층**
```markdown
사용자를 위한 네트워크 응용 환경 지원
```

### 2.1.3. 프로토콜과 인터페이스

- **프로토콜**
```markdown
통신 양단의 각 호스트 간의 계층 n 모듈 간 통신 규칙
```

- **인터페이스**
```markdown
동일한 호스트 내 계층 간 통신 규칙
```

- **서비스**
```markdown
하위 계층이 상위 계층에 제공하는 인터페이스
```

**'호스트1 - 호스트2' 의 계층 n 프로토콜**
```markdown
계층 n 프로토콜이 서로 통신하기 위해서는 계층 n-1 프로토콜의 도움 필요

1. 호스트 1의 계층 n-1에 전송할 데이터 부여
2. 호스트 1의 계층 n-1은 하위 계층으로 호스트 2의 n-1에 전송
3. 호스트 2의 계층 n-1이 받은 데이터를 계층 n으로 전송
```

### 2.1.4. 인터넷 계층 구조

| 호스트 1 | 호스트 2 |
| :-: | :-: |
| 응용 프로그램 클라이언트 | 응용 프로그램 서버 |
| TCP or UDP | TCP or UDP |
| IP | IP |
| LAN 카드 드리아버 | LAN 카드 드라이버 |
```markdown
1. 응용 프로그램 : 5~7 계층
2. TCP or UDP : 4 계층
3. IP : 3 계층
4. 1~2 계층 : 호스트 운영체제에서 구현
```

*양쪽 호스트에는 동일한 기능을 수행하는 **프로토콜 스택**이 존재한다. *
